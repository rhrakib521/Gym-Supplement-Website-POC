01 BRIEF STATEMENT
The Complete Front-End Specification
thryvebd.com requires a full revamp. Not a reskin — a rebuild. The current site is a single-product brochure.
The new site must be a full-stack commercial platform capable of handling 500-5,000 orders per day,
presenting 4-5 product categories, supporting a bilingual audience, and integrating seamlessly with the
Thryve OS backend already specified in Vol 6-10.
Design Direction
The visitor lands and feels two things simultaneously: raw performance energy (intense, athletic, bold) and
Bangladeshi pride (local hero brand, community-first). Not generic supplement SaaS. Not a cheap local copy.
Something that looks like it belongs next to MuscleBlaze — but speaks to Bangladesh specifically.
Inspiration Analysis — What to Take From Each
MuscleBlaze
(muscleblaze.com)
Goal-based navigation (Bulking, Recovery, Fat Loss). Authenticity checker as a trust
anchor page. Lab certificate integration. Subscription toggle directly on product page.
Apparel fully integrated in nav. The product page structure and upsell logic are the closest
reference for Thryve.
BeastLife
(beastlife.in)
Authentication page elevated to navigation-level prominence. Clean Shopify-style
collection pages. Strong "100% Authentic" trust messaging. Category image tiles on
homepage — exactly the structure Thryve needs.
SuperYou
(superyou.in)
Identity-led design. Brand is built around "becoming" someone, not just buying a product.
Clean, minimal, aspirational. The SuperYou tone is the closest to Thryve's Level 2 brand
pivot (aspirational identity). Use this for copy tone and layout whitespace.
Optimum Nutrition
India
Global trust signals. Lab results presented prominently. The product page information
hierarchy is the gold standard — tabs (About, Ingredients, How To Use, Reviews,
Certificate) in that exact order.
Current
thryvebd.com
What to discard: hardcoded testimonials, no account system, no order tracking, no
upselling, single-product focus, no payment gateway integration, no blog, no category
structure. The domain, brand name, and existing customer reviews are the only things to
carry forward.

07 LEAD CAPTURE
Two Methods That Convert in Bangladesh
 Fig 05. Lead capture — WhatsApp sticky bar (primary) and exit intent phone popup (secondary).
Why Phone, Not Email
Email open rates in Bangladesh: 12-18%. WhatsApp open rates: 85-95%. Every phone number captured feeds directly
into the AiSensy lifecycle flows already specified in the Thryve OS (Vol 8). A captured phone number is worth 10x more
than a captured email address in this market.
Method 1 —
WhatsApp Sticky
Bar (Primary)
Fixed green bar at the bottom of every page on both mobile and desktop. Text: "Chat on
WhatsApp · Get ■100 off your first order" + Chat Now button. On click: opens WhatsApp
with pre-filled message. Bot captures phone automatically. Sends discount code. Enters
AiSensy lifecycle flow. Never disappears. Never interrupts. Permanent CTA.
Implementation: simple fixed CSS div + wa.me link with pre-filled text.
Method 2 — Exit
Intent Phone Popup
(Secondary)
Desktop: triggers when cursor moves toward browser close/back button. Mobile: triggers
after 60 seconds of scroll inactivity. Popup: bold offer (■150 off first order), single phone
number field, Bangla + English CTA. Phone only — no name, no email. Single field = 3x
higher completion rate. On submit: AiSensy sends discount code via WhatsApp
immediately. Shows maximum once per session. Dismissed = does not show again for 7
days (cookie).

08 BILINGUAL SPECIFICATION
Bengali + English — From Day One
Both languages live simultaneously. Not a separate Bengali site — a toggled layer. The language toggle is
top-right in the nav. Switching applies to all UI text, CTA copy, error messages, order status labels, and
form placeholders. Product names and brand terms (Thryve, Katana, Mango Ice) stay in English in both
modes.
Language Toggle
Top-right nav button: "■■■■■ / EN". Clicking switches language. State persisted in
localStorage. Detected from browser language preference on first visit. Bangladesh IP +
Bengali browser = Bengali default.
Translation Scope
Navigation labels, page headings, product descriptions, CTA buttons, error messages,
form labels, order status labels, checkout copy, review prompts. Blog posts: authored in
one language, translation optional per post.
Translation Scope —
Excluded
Brand name (THRYVE), product names (Mango Ice, Watermelon Mojito, Katana), athlete
names, scientific terms (Creatine Monohydrate), domain name. These stay in English in
both language modes.

i18n library (react-i18next recommended). Translation JSON files for EN and BN. All UI
strings referenced by key, not hardcoded. Bangla font: Hind Siliguri or Noto Sans Bengali
from Google Fonts. RTL not required — Bengali is LTR.
Forms in Bengali
All form placeholders, error messages, and validation messages in Bengali when Bengali
mode active. Phone field placeholder: "■■■■■ ■■■ ■■■■■ ■■■ (01XXXXXXXXX)".

10 TECHNICAL REQUIREMENTS
Performance · Scale · Backend Integration
The frontend must be built to handle 500-5,000 orders per day. This is not a Shopify theme — it is a custom
React application talking to the Node+Express backend specified in the Thryve OS.
Frontend Stack
React (Next.js recommended for SSR + SEO). Tailwind CSS for styling. react-i18next for
bilingual support. SWR or React Query for data fetching. Deployed on Vercel or Railway.
CDN for all images (existing BunnyCDN setup).
Performance Targets
Lighthouse score: 90+ on mobile. Core Web Vitals: LCP under 2.5s, CLS under 0.1.
Images: WebP format, lazy loading, BunnyCDN delivery. Fonts: preloaded. Critical CSS
inlined. JS bundle split by route.
API Integration
All product data, order data, customer data from Node+Express backend API. No
hardcoded content except static marketing copy. Product catalog: /api/products. Order
placement: /api/orders. Order tracking: /api/track. QR verify: /api/verify/[serial].
Order Volume
Architecture
5,000 orders/day = ~3.5 orders/minute peak. Database: PostgreSQL with connection
pooling (PgBouncer). API: Node cluster mode, horizontal scaling on Railway. Queue:
order confirmation via n8n async — never blocks the HTTP response. CDN: all product
images on BunnyCDN, never from origin on each request.
Backend Integration
Points
Website places order → Node API → PostgreSQL + Nuport. WhatsApp confirmation (Flow
2) triggered by n8n webhook. QR serial lookup → PostgreSQL qr_serials table.
Subscription toggle → Node subscription endpoint. Lead capture → AiSensy API directly
from Node backend. Payment: SSLCommerz redirect, bKash/Nagad inline OTP via their
APIs.
Dark/Light Mode
Both modes supported. CSS custom properties for all colours. User preference persisted
in localStorage. System preference detected on first visit. Dark: background #0D1117,
accent #1D9E75. Light: background white, accent #1D9E75. Accent colour (Thryve green)
remains consistent across both modes.
SEO Requirements
Next.js SSR for product and blog pages (Google must index them). Meta titles and
descriptions per page. Open Graph images for social sharing. Structured data: Product
schema, BreadcrumbList, Organization. Bengali page titles and descriptions when Bengali
mode active. Sitemap auto-generated. robots.txt allowing all crawlers.
Accessibility
WCAG 2.1 AA minimum. All images have alt text. Form inputs have visible labels. Colour
contrast ratio 4.5:1 minimum. Keyboard navigable. Screen reader compatible for all
interactive elements.

11 UPSELLING + CONVERSION LOGIC
Every Touchpoint That Increases AOV
Average Order Value is ■2,175. Target: ■2,800+. Three upsell surfaces increase AOV without a dedicated upsell
page.
Product Page —
Upsell Widget
Amber-bordered card below the Add to Cart button. "Frequently Bought Together: Add
[complementary product] for ■200 off." One-tap to add bundle to cart. Logic powered by
cross-sell rules in the Upsell Timing Agent. Rule example: Mango Ice buyer → show
Watermelon bundle.
Cart Page —
Suggested Add
Below cart items: "You might also need:" — one accessory (shaker or bag). Shown only if
cart contains only supplement, no accessories. Simple rule-based, not ML. ■800 average
accessory price lifts AOV significantly.
Checkout Step 3 —
Last Upsell
"Before you confirm: add [product] for ■X off — limited to this order." One-click add. Does
not leave checkout flow. Should be a consumable: a sample sachet of pre-workout
(coming soon), or a second creatine jar at ■200 off.
Post-Purchase Page
"Add to your order before it ships." One product shown with a 15-minute countdown timer.
If clicked: appended to the same Nuport order before dispatch. After 15 minutes: offer
expires, standard price shown.
Subscription Pitch
On Day 30 reorder message (lifecycle Flow 8) and on the subscription toggle in the
product page. "Subscribe: ■1,999/month. Save 13%. Never run out." Subscription AOV =
lower per unit but higher LTV. Primary focus once base is built.
